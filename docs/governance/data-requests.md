# Data requests

The board includes tasks on standardising the data request procedure and developing and publishing sample and data request forms.

## Goal

Create a request process that is clear for researchers and manageable for the team.

## Minimum request workflow

Step-by-step detail: [Data request workflow](../workflows/data-request-workflow.md). Summary:

1. Receive request form.
2. Check completeness.
3. Confirm scientific and governance fit.
4. Confirm requested data or sample scope.
5. Prepare extract or response.
6. Record what was shared and when.

## Minimum form contents

A request form should capture:

- applicant and institution
- project title and purpose
- requested data categories or variables
- requested sample types if applicable
- ethical approval status where relevant
- data transfer and access arrangements
- expected outputs or publication plan

## Internal review points

Before release, confirm:

- the requested data exists
- the request is in scope
- identifiers and anonymisation level are appropriate
- the deliverable can be reproduced if asked again later

## Sample requests

Sample requests follow the same high-level pattern as data requests. In addition:

- use [OpenSpecimen](../systems/openspecimen.md) for sample tracking
- coordinate with biobank for retrieval and transfer
- follow the [Sample request workflow](../workflows/sample-request-workflow.md)

## Suggested deliverables to maintain

- public-facing data request form
- public-facing sample request form
- internal review checklist
- request log (data and sample)
- standard email templates for approval, revision, and rejection
