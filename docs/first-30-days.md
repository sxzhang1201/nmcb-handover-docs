# First 30 days

This page is a practical onboarding sequence for the next person taking over the role.

## Week 1

- Review the [data lifecycle overview](lifecycle/index.md) to understand the pipeline.
- Obtain access to all core systems and shared folders.
- Review the regular task rhythm.
- Confirm where the current source of truth is for participant identifiers.
- Review the latest task board and any open blockers.

## Week 2

- Run through one full cycle of the weekly tasks.
- Validate the current Castor export and reminder workflow.
- Review at least one device QC example for each major device workflow.
- Check script locations, dependencies, and versioning status.

## Week 3

- Review Snowflake or SQL-related setup and current access limitations.
- Walk through how a data request would be processed end to end.
- Review the biobank data flow and multicentre transfer dependencies.

## Week 4

- Update this documentation where reality differs from the written process.
- Create a short list of immediate risks and quick wins.
- Confirm who approves changes in operational workflows, code, and governance documents.

## Minimum access checklist

The next person should confirm access to:

- shared mailbox
- participant tracking files
- Castor EDC
- device output locations
- script repository or folders
- Snowflake or SQL environment
- documentation folders and SOP locations
