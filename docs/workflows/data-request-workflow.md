# Data request workflow

Procedure from researcher request to **secure delivery** of study data. Governance context and form contents are summarised in [Data requests (governance)](../governance/data-requests.md).

## Goal

A process that is clear for researchers and manageable for the team, with traceability and reproducibility.

## Minimum workflow

1. **Receive** request (form or equivalent).
2. **Check** completeness.
3. **Confirm** scientific and governance fit (in scope, approvals).
4. **Confirm** requested data or variable scope against the codebook / Snowflake.
5. **Prepare** extract or response (scripts, queries, MyDRE or agreed channel).
6. **Record** what was shared, to whom, and when (request log).

## Internal review before release

- Requested data exists and is correct for the ask.  
- Request is in scope.  
- Identifiers and anonymisation level are appropriate.  
- The deliverable can be reproduced if asked again later.

## Delivery channels

Define and document:

- when delivery is via **myDRE** vs file transfer vs other approved route  
- rules for Excel or file exports vs query-only access  

## Suggested artefacts to maintain

- public or internal **data request form**  
- **internal review checklist**  
- **request log**  
- standard email templates for approval, revision, and rejection
